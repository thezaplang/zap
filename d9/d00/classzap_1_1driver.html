Detailed Description

Orchestrates and controls the entire compilation process. Order of the functions that should be called is by how they are defined here in the header. Meaning that first is parseArgs(int, char**), second is splitInputs(), third is verifySources(), and so on... Most functions return true on error not on success. It returns error due to easy checks like if(verifyOutput()) return 1;.
